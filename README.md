# AI 기반 데이터 이상치 탐지 프로그램 (단일 HTML)

내부망(오프라인) 환경을 위한 **설치가 전혀 필요 없는** 데이터 이상치 탐지·비교 검증·개인정보 비식별·데이터 정제/변환 도구입니다.
**`index.html` 파일 하나**로 동작하며, 웹 브라우저로 열기만 하면 실행됩니다. (현재 버전: **v2.1**)

- **설치 불필요**: 파이썬·Node·npm·pip 아무것도 필요 없습니다.
- **외부 라이브러리 미사용**: CDN·인터넷 통신 없이 순수 JavaScript 로만 구현.
- **오프라인 100%**: 모든 처리가 브라우저 내부에서 이루어지며 데이터가 외부로 전송되지 않습니다.

> XLSX, XLS, CSV — 결측치 · 중복 · 범위 초과 · 증감률 · IQR · Z-Score · Isolation Forest · LOF · DBSCAN · 두 파일 비교 검증 · 개인정보 비식별 처리 · 데이터 정제 및 형식 변환

화면 **왼쪽 레일**에서 네 가지 모드 — **단일 파일 분석 · 비교 검증 · 개인정보 비식별 · 정제·형식 변환** — 를 전환합니다.
(좁은 화면 ≤720px 에서는 상단 탭으로 자동 전환됩니다.)

---

## 실행 방법

1. `index.html` 파일을 더블클릭합니다. (또는 브라우저로 파일 열기)
2. 끝입니다. 별도 설치나 서버가 필요 없습니다.

권장 브라우저: 최신 **Chrome / Edge / Firefox**.

## ① 단일 파일 분석

1. **파일 선택** — 버튼 또는 **Drag & Drop** 으로 csv / xlsx / xls 파일을 올립니다.
   업로드가 끝나면 업로드 영역에 **파일명이 표시**되고, [✕ 파일 제거] 로 초기화할 수 있습니다.
2. **분석 옵션 / 파라미터** — 옵션명에 마우스를 올리면(모바일은 터치) 의미·권장값·주의사항을 담은 **도움말**이 표시됩니다.
   설정 패널은 접이식이며, 해당 기능의 옵션을 모두 끄면 관련 설정 패널이 자동으로 접힙니다.
3. **숫자 컬럼 설정** — 자동 추출된 숫자 컬럼의 허용 최소/최대를 지정합니다.
4. **분석 시작** — 분석은 **Web Worker(백그라운드)** 에서 실행되어 대용량에서도 화면이 멈추지 않습니다.
   분석 결과 요약의 히스토그램은 **마우스 휠로 확대/축소**하여 값을 세밀하게 볼 수 있습니다.
5. **결과 다운로드** — 원본 + `이상치 여부`/`이상치 사유`/`판정 근거`/`검토 상태`/`검토 메모` 컬럼.
   - **원본이 xlsx 인 경우**: 원본의 **서식(글꼴·글씨크기·볼드·색상)과 열 너비·행 높이를 그대로 보존**한 채
     분석 컬럼만 오른쪽에 덧붙여 저장합니다(이상 행의 덧붙인 셀은 노란색 강조).
   - csv/xls 또는 보존 실패 시: `Result`/`Anomaly` 2시트로 저장(이상 행 노란색 #FFF2CC). 날짜 값은 실제 날짜 셀로 저장됩니다.

### 탐지 항목

| 구분 | 검사 | 비고 |
|------|------|------|
| 항상 수행 | **형식 오류** | 숫자 컬럼에 섞인 문자 값(예: `"abc"`)을 별도 사유로 표시 |
| 규칙 | 결측치 / 중복 / 범위 / 증감률 | 증감률 임계치(%) 조절 |
| 통계 | IQR(배수 1.5) / Z-Score(임계치 조절) | |
| 머신러닝 | Isolation Forest / LOF / DBSCAN | 순수 JS 구현. **LOF·DBSCAN 은 2만 행 초과 시 자동 건너뜀 후 안내** |

수행되지 못한 검사가 있으면 결과 화면에 **건너뜀 사유가 표시**됩니다.

## ② 두 파일 비교 검증

상단 **[두 파일 비교 검증]** 탭에서 사용합니다.

1. **파일 A(기준·이전)** 와 **파일 B(비교·이후)** 를 올립니다. CSV↔XLSX 조합도 가능합니다.
2. **기준 컬럼 자동 추론** — 이름 유사도(동의어: 산단명↔산업단지명, 업체명↔회사명 등) +
   고유값 비율 + 값 겹침을 종합해 가장 알맞은 컬럼 쌍을 추천합니다. 직접 변경도 가능합니다.
3. **이상 변동 기준** — 변화율(%) / Z-Score / IQR 를 선택합니다.
   Z-Score·IQR 은 매칭된 모든 행의 변화량(변화 없음=0 포함)을 모수로 계산합니다.
4. **비교 시작** — 결과가 아래와 같이 분류됩니다.

| 분류 | 설명 |
|------|------|
| 추가 / 삭제 | B에만 있거나 A에만 있는 행 (행 내용 스냅샷 표시) |
| 변경 | 값이 달라진 행 — 셀별 `이전 → 이후 (+50%)` 강조 |
| 이상 변동 | 급격한 증가/감소, 통계적으로 유난히 큰 변화 **및 결측값 변경** |
| 결측값 변경 | 값↔결측 전환 — **이상 변동으로도 함께 판정됩니다** |
| 타입 변경 | 숫자↔문자, 날짜↔문자 전환 |

요약(총 행/동일/변경/추가/삭제/이상 변동/결측값 변경/타입 변경)과 필터를 제공하며,
**[비교 결과 다운로드]** 로 상태별 색상이 적용된 Excel(Compare/Summary 2시트)을 받을 수 있습니다.

## ③ 개인정보 비식별 처리

왼쪽 레일 **[개인정보 비식별]** 에서 사용합니다. 개인정보가 포함된 표를 올리면
컬럼별 유형을 자동 추정하고, 선택한 방식으로 비식별 처리한 파일을 내려받을 수 있습니다.

1. **파일 업로드** — csv / xlsx / xls.
2. **컬럼별 유형 자동 추정** — 컬럼명 사전 + 값 패턴(정규식) 매칭률을 종합해 추정합니다.
   지원 유형: 주민등록번호·외국인등록번호·여권·운전면허·휴대전화·유선전화·이메일·
   카드번호(Luhn 검증)·계좌번호·사업자등록번호·법인등록번호·생년월일·주소·IP·성명.
   *산단명·공장명·기업명·업체명* 등 사업체/시설 명칭은 개인정보가 아니므로 탐지에서 제외되며,
   사업자등록번호는 맥락 의존성이 높아 신뢰도를 '중간' 으로 표시합니다.
3. **처리 방식 선택** — 컬럼별로 다음 중 선택하고 [적용]을 체크합니다.

   | 방식 | 설명 |
   |------|------|
   | 마스킹 | 일부만 노출(예: `홍**`, `010-****-5678`, `a***@ex.com`) |
   | 삭제 | 값 제거 |
   | 가명 | 동일 원본을 동일 가명(`익명1`, `익명2` …)으로 일관 치환 |
   | 일반화 | 생년월일→연대(예: `1990년대`), 주소→시·도/시군구 수준 |
   | 해시 | 순수 JS **SHA-256**(솔트 필수) — 외부/WebCrypto 의존 없음 |

   기본 매핑: 주민/외국인/여권/면허=삭제, 성명/전화/이메일/카드/계좌=마스킹, 생년월일/주소=일반화.
4. **미리보기** — 상위 20행의 `원본 → 변환` 을 확인합니다.
5. **다운로드** — `비식별결과`/`처리요약` 2시트 Excel. 수식(`=`,`@`) 무력화 규칙이 동일하게 적용됩니다.

## ④ 데이터 정제 · 형식 변환

왼쪽 레일 **[정제·형식 변환]** 에서 사용합니다. 공공데이터포털 등록을 고려한 **데이터 정제 + 파일 형식 변환**을
여러 파일에 대해 한 번에 처리합니다.

### 변환 방식 (4종)

| 방식 | 설명 |
|------|------|
| XLSX → XLSX | 정제 후 **원본 형식·서식 그대로 저장**(원본 서식 유지) |
| XLSX → CSV | 시트별로 CSV 생성(다중 시트는 파일별로 분리) |
| CSV → CSV | 정제 후 CSV로 저장 |
| CSV → XLSX | CSV를 엑셀(xlsx)로 변환 |

### 데이터 정제 옵션

기본으로 켜져 있는 **권장 정제**(해제 가능): 앞뒤 공백 · 연속 공백 · 줄바꿈 · Tab · 제어문자 · HTML 태그 ·
보이지 않는 유니코드 · 컬럼명 공백/특수문자 정리.

**특수문자 처리 방식**(라디오, 기본 '일반만 제거'):
- 유지 / **일반적인 특수문자만 제거**(`! @ # $ ^ & * ` ~ | < > { } [ ]` 제거) / 모든 특수문자 제거
- 유지 대상: 한글 · 영문 · 숫자 · `-` `_` `/` `.` `()` `%` `+` — (i) 아이콘에서 제거/유지 대상을 안내합니다.

**선택 옵션**(기본 꺼짐): 이모지 제거 · 빈 문자열↔NULL · **`"-"` → 0 치환**
(숫자로 인식되는 컬럼의 `-`(결측 표기)만 0으로 바꾸며, 코드값·문자열의 `-`는 보존합니다).

### 다중 파일 · 인코딩 · 결과

- **다중 업로드**(드래그&드롭/선택, 최대 30개·총 300MB), 순차 처리로 브라우저가 멈추지 않습니다.
- 파일별 성공/실패 표시 + 전체 진행률 + 처리 요약(제거된 특수문자/제어문자 수, 정리된 컬럼명 수, 변환 파일/시트 수).
- 출력이 여러 개면 **ZIP으로 한 번에** 내려받습니다.
- **한글 깨짐 방지**: CSV 저장은 기본 **UTF-8 (BOM)**, 불러오기는 **UTF-8 BOM → UTF-8 → CP949(EUC-KR)** 순으로 자동 판별(수동 지정도 가능).

## 기술적 참고 (외부 라이브러리 없이 구현한 부분)

| 기능 | 구현 방식 |
|------|-----------|
| xlsx 읽기 | ZIP 파싱 + **순수 JS DEFLATE 압축 해제(tiny-inflate 포팅)** + XML 파싱(DOMParser) |
| xls 읽기 | **OLE2(CFB) 컨테이너 해석** + BIFF8 레코드 파서, 셀 서식(XF/FORMAT/1904체계) 기반 날짜 변환 |
| csv 읽기 | 따옴표/개행 처리 + **UTF-8 BOM → UTF-8 → CP949(EUC-KR)** 자동 판별(강제 지정 가능), 청크 파싱(진행률) |
| csv 쓰기 | RFC4180 인용 + **UTF-8 BOM 기본**(CsvWriter) — Windows Excel/LibreOffice 한글 정상 표시 |
| Excel 쓰기 | ZIP(STORED) 직접 생성 + CRC32, 노랑/초록/빨강 채움, 날짜 셀(numFmt) |
| Excel 서식 보존(덧붙임) | 원본 xlsx ZIP 엔트리(styles/글꼴/열너비/행높이/테마) 유지 + 대상 시트에 분석 컬럼 덧붙임(XlsxAppender) |
| Excel 서식 보존(정제) | 원본 셀은 그대로 두고 **텍스트 셀 값만** 정제, 스타일(`s`)·숫자/날짜/수식 셀 보존(XlsxCleaner) |
| 공통 다운로드 | 모든 저장(xlsx/csv/zip)이 단일 모듈(Downloader)을 경유 — 프로젝트 전체 동일 로직 |
| 데이터 정제 | 공백/제어문자/HTML/유니코드/특수문자·컬럼명 정리, `"-"→0`(숫자 컬럼만) 등(DataCleaner, DOM 미사용) |
| 분석 실행 | 인라인 **Web Worker**(불가 환경은 메인 스레드 폴백) |
| 알고리즘 | IQR/Z-Score 및 Isolation Forest/LOF(메모리 O(n·k))/DBSCAN 전부 vanilla JS |
| 개인정보 비식별 | 컬럼명 사전 + 값 정규식 매칭 기반 유형 추정, **순수 JS SHA-256**(외부/WebCrypto 미의존) |

### 보안
- 컬럼명/셀 값은 화면 표시 전 HTML 이스케이프 처리(XSS 방어).
- 확장자 위조는 매직바이트로 차단, 압축 해제 크기 상한(zip-bomb 방어).
- 내려받는 Excel 에서 `=`, `@` 로 시작하는 문자열은 **작은따옴표(') 접두로 무력화**되어
  수식으로 실행되지 않습니다(CSV Injection 방어).

### 알려진 한계
- LOF·DBSCAN 은 2만 행 초과 데이터에서 자동으로 건너뜁니다(결과 화면에 안내).
- 분석/비식별 파일 업로드 최대 50MB. 정제·형식 변환은 다중 파일 최대 30개·총 300MB.

## 파일 구성

```
index.html                  # 프로그램 본체 (이 파일 하나면 실행 가능)
sample_data/                # 테스트용 샘플 22개 + 안내 README
scripts/generate_samples.py # 샘플 생성 스크립트(개발용, 실행에는 불필요)
tests/                      # 헤드리스 브라우저 회귀 테스트(Playwright) + 실행 안내
README.md
```
